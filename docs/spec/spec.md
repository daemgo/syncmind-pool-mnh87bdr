## 1. 产品概述

### 1.1 产品定位

长机科技 EAM 系统是一款面向装备制造企业的设备全生命周期管理平台。系统以设备台账为基础，以预防性维护为核心，以监控大屏为窗口，帮助企业从"被动维修"走向"主动管理"，从"经验驱动"转向"数据驱动"。

### 1.2 核心价值

- **台账清晰**：为每一台设备建立唯一数字身份档案，设备从进厂到报废的全生命周期信息可查、可追溯。
- **维护前置**：基于运行数据和维护计划提前发现隐患，将故障消灭在萌芽阶段，减少意外停机。
- **状态可见**：监控大屏实时呈现设备运行状态，管理层和车间主管随时掌握全局。
- **数据贯通**：打通西门子数控系统的数据孤岛，让设备数据真正服务于管理决策。

### 1.3 目标用户与使用场景

| 用户角色 | 核心使用场景 |
|---------|------------|
| 设备管理员 | 录入和维护设备台账，制定预防性维护计划，管理备品备件 |
| 维修人员 | 接收工单，执行维修任务，记录维修过程和结果 |
| 车间主管 | 通过监控大屏实时查看车间设备状态，调度工单，处置异常 |
| 管理层 | 通过监控大屏和报表了解设备整体运行状况，基于数据做决策 |

### 2.2 本期不含

| 功能 | 不含原因 |
|-----|---------|
| **状态监测预警**（基于温度、振动、负载的实时告警） | 属于工业 IoT 范畴，需额外传感器硬件部署和数据建模，超出本期预算和工期范围 |
| **移动端应用**（移动端巡检、报修、工单处理） | 方案中标注 P3 优先级，属于二期扩展内容；本期聚焦核心管理功能 |
| **预测性维护**（基于 AI 的故障预测） | 需要足够的设备运行数据积累，本期先建立基础数据和预防性维护体系 |
| **与其他 ERP/MES 系统集成** | 本期以西门子系统集成为主，其他系统待后续评估 |
| **多工厂/多基地支持** | 长机科技本期为单一工厂场景，架构上预留扩展性 |

### 2.3 边界说明

- **数据采集范围**：本期仅对接西门子数控系统的设备运行数据，不涉及其他品牌设备的数据接入。
- **备件管理边界**：备品备件管理本期仅覆盖常用备件的库存台账和预警，暂不涉及采购流程管理。
- **报表时间范围**：分析报表基于系统上线后的数据进行展示，历史数据（上线前）根据实际整理情况酌情导入。

## 3. 功能设计

### 3.1 设备台账管理

#### 3.1.1 功能概述

为每台设备建立唯一的数字档案，记录设备从进厂到报废的全生命周期信息。设备管理员在此模块完成设备信息的登记和维护，是整个 EAM 系统的数据基础。

#### 3.1.2 解决的问题

设备信息分散在老工程师的脑海里和 Excel 表格中，新人接手找不到资料，故障历史无从追溯。台账上线后，任何人打开系统就能看到这台设备的所有信息。

#### 3.1.3 业务规则

- 设备编号在全厂范围内唯一，不允许重复。
- 设备新增时必须填写：设备编号、设备名称、规格型号、所属车间、安装日期、生产厂商。
- 设备状态枚举：`运行中`、`停机待修`、`停机待料`、`报废`、`闲置`。
- 设备每次发生状态变更时，系统自动记录变更时间和操作人，形成生命周期时间轴。
- 设备关联西门子系统集成点编号，实现与数据采集系统的绑定。
- 设备档案支持上传附件（如设备说明书、现场照片），附件格式支持 PDF、图片，单个文件不超过 20MB。

#### 3.1.4 列表设计

**列表类型**: 主列表
**排序默认**: 按设备编号升序
**筛选字段**: 所属车间、设备状态、规格型号、关联西门子系统编号

| 列名 | 字段 ID | 列类型 | 宽度 | 排序 |

| 设备编号 | assetCode | 文本 | 120px | 可排 |
| 设备名称 | assetName | 文本 | 200px | 可排 |
| 所属车间 | workshop | 枚举 | 100px | 可排 |
| 规格型号 | model | 文本 | 150px | 不可 |
| 设备状态 | status | 枚举 | 100px | 可排 |
| 最近维护日期 | lastMaintenanceDate | 日期 | 120px | 可排 |
| 操作 | - | 操作列 | 80px | 固定 |

**操作项**: 查看详情、编辑

#### 3.1.5 字段定义

| 字段 ID | 字段名称 | 字段类型 | 必填 | 约束/规则 |
|---------|---------|---------|-----|---------|
| assetCode | 设备编号 | 文本 | 是 | 全厂唯一，支持字母+数字，自动校验重复 |
| assetName | 设备名称 | 文本 | 是 | 最多 100 字符 |

设备生命周期事件自动记录，包含以下事件类型：`创建`、`状态变更`、`维修记录`、`维护计划变更`、`报废`。

生命周期时间轴以卡片形式展示，每条记录显示：事件类型、发生时间、操作人、变更内容摘要。

### 3.2 预防性维护

#### 3.2.1 功能概述

基于设备类型、运行时间和历史故障数据，制定周期性维护计划，系统自动在计划时间触发维修工单，推动维护工作从被动响应走向主动预防。

#### 3.2.2 解决的问题

以前维护靠老工程师"看情况"，设备什么时候该保养全凭经验。预防性维护上线后，系统会按时提醒，该换油换油、该检查检查，设备故障率自然下来。

#### 3.2.3 业务规则

- 维护计划按设备维度创建，一个设备可以有多条维护计划。
- 维护周期枚举：`每日`、`每周`、`每月`、`每季度`、`每半年`、`每年`、`按运行小时数`。
- 按运行小时数触发时，需设定触发阈值（如每运行 2000 小时），阈值数据来源于西门子系统集成。
- 计划执行后，系统自动生成对应的维修工单，工单状态为`待派工`。
- 维护计划支持设定"提前提醒天数"，在计划日期前 N 天在监控大屏提醒区域展示。
- 维护计划执行后，更新该设备的上次维护日期，用于下次计划计算。
- 维护计划可关联标准作业指导书（以附件形式上传），维修人员执行时有参考。

#### 3.2.4 列表设计

**列表类型**: 主列表
**排序默认**: 按下次维护日期升序（临近的排前面）
**筛选字段**: 所属车间、设备编号、维护周期、计划状态

| 列名 | 字段 ID | 列类型 | 宽度 | 排序 |
|-----|---------|-------|------|------|
| 设备编号 | assetCode | 文本 | 120px | 可排 |
| 设备名称 | assetName | 文本 | 200px | 不可 |
| 维护内容 | content | 文本 | 200px | 不可 |
| 维护周期 | cycle | 枚举 | 100px | 不可 |
| 下次维护日期 | nextMaintenanceDate | 日期 | 120px | 可排 |
| 计划状态 | planStatus | 枚举 | 100px | 可排 |
| 操作 | - | 操作列 | 80px | 固定 |

**操作项**: 查看详情、编辑、立即执行

**枚举值 - 计划状态**: `执行中`、`已暂停`、`已到期未执行`

#### 3.2.5 字段定义

| 字段 ID | 字段名称 | 字段类型 | 必填 | 约束/规则 |
|---------|---------|---------|-----|---------|
| assetId | 关联设备 | 关联 | 是 | 关联设备台账中的设备 |
| content | 维护内容 | 多行文本 | 是 | 最多 300 字符，描述本次维护的具体事项 |
| cycle | 维护周期 | 枚举 | 是 | 枚举值见业务规则 |

| nextDate | 下次维护日期 | 日期 | 条件必填 | 当 cycle 非`按运行小时数`时必填 |

| sopAttachment | 标准作业指导书 | 附件 | 否 | PDF 格式优先，单文件≤50MB |
| planStatus | 计划状态 | 枚举 | 是 | 默认值`执行中` |

### 3.3 维修工单管理

#### 3.3.1 功能概述

覆盖工单从创建到归档的全流程闭环管理。工单可以由设备故障报修触发，也可以由预防性维护计划自动生成。维修人员接收工单、执行任务、记录过程，最终由设备管理员或车间主管验收确认。

#### 3.3.2 解决的问题

以前维修靠打电话，工单状态没人清楚，活干完了可能也没人知道。工单管理上线后，每个工单都有明确的责任人，流程走到哪一步系统里清清楚楚，维修知识也能积累下来。

#### 3.3.3 业务流程

工单状态流转：`创建 → 待派工 → 待执行 → 执行中 → 待验收 → 已完成/已归档`

- **创建**：设备管理员或系统（预防性维护）创建工单。
- **派工**：车间主管或设备管理员将工单派给具体维修人员。
- **执行**：维修人员执行维修任务，填写执行记录和实际使用的备件。
- **验收**：设备管理员或车间主管确认维修结果。
- **归档**：验收通过后工单归档，归档后不可修改。

#### 3.3.4 业务规则

- 工单来源枚举：`故障报修`、`预防性维护`、`计划检修`、`其他`。
- 工单优先级枚举：`紧急`、`高`、`中`、`低`。紧急工单在列表中置顶显示，监控大屏实时预警区同步高亮。
- 工单创建时关联设备，自动带出设备编号和所属车间。
- 派工时必须指定执行人，执行人可以为一人或多人（多人时需指定主执行人）。
- 执行记录支持多次填写，每次填写记录时间和执行内容，形成执行日志。
- 备件消耗从备品备件库存中扣减，库存不足时提示但允许手动确认出库。
- 工单验收时可对本次维修评分（1-5星），评分数据用于分析报表。
- 验收时需填写故障原因和处理措施，作为维修知识积累。
- 同一设备在维修中的工单数量超过 2 个时，系统提示设备频繁故障。
- 工单列表默认展示`执行中`和`待派工`状态的工单，已归档工单通过筛选查看。

**列表类型**: 主列表

### 3.4 备品备件管理

#### 3.4.1 功能概述

建立备品备件库存台账，将备件与设备BOM（物料清单）关联，设置安全库存和预警规则。维修工单执行时消耗备件，系统自动扣减库存，库存不足时触发预警。

#### 3.4.2 解决的问题

以前车间有什么备件、够不够用全靠人记。备件管理上线后，库存一目了然，缺什么系统会提醒，维修的时候也能知道该用什么备件、从哪里领。

#### 3.4.3 业务规则

- 备件编号和备件名称组合唯一。
- 备件分类枚举：`机械类`、`电气类`、`液压类`、`刀具类`、`其他`。
- 安全库存为最低库存量，当实际库存低于安全库存时，系统在监控大屏预警区展示提示。
- 当前库存 + 在途库存 < 安全库存时，触发补货提醒。
- 备件与设备通过BOM关联，一个设备可以关联多个备件。
- 工单执行时从备件库存中扣减数量，库存不足时允许手动确认继续出库，但需填写说明。
- 备件入库时记录入库时间、来源（采购/维修退回/其他）和数量。
- 库存数据按批次管理，最早入库的批次优先消耗（FIFO原则）。

#### 3.4.4 列表设计

**列表类型**: 主列表
**排序默认**: 备件编号升序

### 3.5 监控大屏

#### 3.5.1 功能概述

在车间或管理办公室部署监控大屏，实时展示设备运行状态、OEE指标、故障预警等关键信息。大屏是管理层和车间主管最常用的信息窗口，也是整个 EAM 系统"状态可见"这一核心价值的直接体现。

#### 3.5.2 解决的问题

以前设备开没开机、有没有故障，车间主任得亲自走过去看，或者等生产人员打电话汇报。大屏上线后，站在大屏前就能看到全厂设备状态，有问题第一时间知道，不用等电话。

#### 3.5.3 布局设计

**布局类型**: 固定5区布局
**大屏尺寸适配**: 支持 1920×1080（55-65寸）到 3840×2160（98寸）的分辨率自适应，所有元素按比例缩放。

| 区块 | 区块类型 | 内容说明 |
|-----|---------|---------|
| 全局状态区 | 指标卡区块 | 设备总览（总数/运行中/停机/故障）、综合OEE |
| 分区状态区 | 图块/列表区块 | 按车间/产线展示设备分布和运行状态，颜色编码：绿色=运行、灰色=停机、红色=故障 |
| 关键指标区 | 图表区块 | TOP5故障设备、TOP5低效设备、维护计划执行率 |
| 实时预警区 | 告警列表区块 | 最新故障预警（红色闪烁）、异常设备列表、待处理工单（紧急工单高亮） |
| 趋势分析区 | 折线图区块 | OEE趋势（7天/30天切换）、故障率趋势、维护成本趋势 |

**告警规则**：

- 设备状态=`故障`时，设备图标红色闪烁，刷新间隔 5 秒。
- 紧急工单在高亮提醒，停留 30 秒后进入常规列表。
- 库存低于安全库存时，备件名称以橙色文字展示。
- 维护计划到期前 N 天（按提前提醒天数配置），在大屏预警区展示。

#### 3.5.4 技术特性

- 数据刷新延迟≤5秒。
- 支持大屏硬件从 55 寸到 98 寸，分辨率自适应。
- 异常状态红色闪烁告警，支持声音提示（可配置开关）。
- 支持自定义布局和指标配置（管理员可在后台调整各区块展示的指标和排序规则）。

### 3.6 西门子系统集成

#### 3.6.1 功能概述

与西门子数控系统对接，实时采集设备运行数据和状态信息，为预防性维护、OEE分析和监控大屏提供数据来源。本期以集成对接为主，建立数据采集通道。

#### 3.6.2 解决的问题

西门子系统里有大量设备运行数据，但以前这些数据躺在系统里没人用。现在打通之后，设备什么时候开机、运行了多久、有没有异常，系统里都能看到，为后续的数据分析打好基础。

#### 3.6.3 业务规则

- 每个西门子集成点编号与设备台账中的设备一一对应。
- 采集的数据项：`运行状态`（运行中/停机）、`运行时间累计`（小时）、`故障代码`。
- 数据采集频率：运行状态每 30 秒采集一次，运行时间每小时更新一次。
- 集成异常时（如西门子系统连接中断），监控大屏显示对应设备"数据采集异常"标识，不影响其他设备展示。
- 西门子系统的故障代码与设备台账中的故障类型做映射，映射关系由管理员在后台配置。

#### 3.6.4 集成点配置

| 字段 ID | 字段名称 | 字段类型 | 必填 | 约束/规则 |
|---------|---------|---------|-----|---------|
| integrationCode | 集成点编号 | 文本 | 是 | 对应西门子系统中的设备标识，全厂唯一 |
| assetId | 关联设备 | 关联 | 是 | 关联设备台账中的设备 |
| systemType | 系统类型 | 枚举 | 是 | 本期固定`西门子数控系统` |
| endpoint | 接口地址 | 文本 | 是 | 西门子系统的数据接口 URL |
| status | 连接状态 | 枚举 | 是 | 系统自动维护：`正常`、`异常`、`未连接` |
| lastSyncTime | 最后同步时间 | 日期时间 | 是 | 系统自动记录 |

**枚举值 - 连接状态**: `正常`、`异常`、`未连接`

### 3.7 分析报表

#### 3.7.1 功能概述

提供设备OEE、故障率、维护成本等多维度的分析报表，帮助管理层了解设备整体运行状况，发现问题持续改进。

#### 3.7.2 解决的问题

以前汇报设备情况靠人工统计数字，报表出得慢、数据还不一定准。报表上线后，系统自动从设备台账、工单记录、备件消耗中取数生成报表，管理层随时能看到最新的设备运行分析。

#### 3.7.3 报表类型

**OEE分析报表**

| 字段 ID | 字段名称 | 字段类型 | 说明 |
|---------|---------|---------|------|
| assetCode | 设备编号 | 文本 | 分组维度 |
| assetName | 设备名称 | 文本 | 分组维度 |
| availability | 可用率 | 百分比 | 运行时间 / 计划运行时间 |
| performance | 性能率 | 百分比 | 实际产量 / 理论产量（本期暂不支持性能率精确计算，默认展示 `--`） |
| quality | 质量率 | 百分比 | 本期暂不支持，默认展示 `--` |
| oee | OEE | 百分比 | 本期简化为：可用率 × 性能率默认值 0.9 × 质量率默认值 0.95 |

- 展示维度：按设备、按车间
- 时间范围：本周、本月、本季度、自定义
- 图表类型：柱状图（按设备）、折线图（按时间趋势）

**故障率分析报表**

| 字段 ID | 字段名称 | 字段类型 | 说明 |
|---------|---------|---------|------|
| assetCode | 设备编号 | 文本 | 分组维度 |
| assetName | 设备名称 | 文本 | 分组维度 |
| faultCount | 故障次数 | 整数 | 统计期内故障工单数量 |
| totalDowntime | 总停机时长(小时) | 数值 | 统计期内累计停机时长 |
| mtbf | MTBF(小时) | 数值 | Mean Time Between Failures = 统计期运行小时 / 故障次数 |
| mttr | MTTR(小时) | 数值 | Mean Time To Repair = 总停机时长 / 故障次数 |

- 展示维度：按设备、按车间
- 时间范围：本周、本月、本季度、自定义
- 图表类型：柱状图（按设备故障次数降序）、折线图（按月趋势）

**维护成本分析报表**

| 字段 ID | 字段名称 | 字段类型 | 说明 |
|---------|---------|---------|------|
| assetCode | 设备编号 | 文本 | 分组维度 |
| assetName | 设备名称 | 文本 | 分组维度 |
| laborCost | 人工成本(元) | 数值 | 本期按工单数量估算，单工单基准费用由管理员配置 |
| materialCost | 备件成本(元) | 数值 | 工单关联的备件出库金额 |
| totalCost | 总维护成本(元) | 数值 | 人工成本 + 备件成本 |

- 展示维度：按设备、按车间
- 时间范围：本周、本月、本季度、自定义
- 图表类型：饼图（成本构成）、柱状图（按设备成本降序）

#### 3.7.4 业务规则

- 报表数据基于系统上线后的数据进行计算，历史数据根据实际情况酌情导入。
- 报表支持导出为 Excel 格式。
- OEE 报表的性能率和质量率本期无法精确计算，页面展示 `--` 并注明"本期性能率/质量率暂不支持精确计算"。
- 所有报表默认时间范围为`本月`。
